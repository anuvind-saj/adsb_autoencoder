# requirements_rpi.txt — Headless Linux ARM64 (Raspberry Pi 4/5, 64-bit OS)
# ============================================================================
# Install on the Pi with:
#   pip install -r requirements_rpi.txt
#
# Notes
# -----
# • --index-url points pip at PyTorch's dedicated CPU-only wheel index.
#   That index carries explicit +cpu aarch64 wheels (e.g.
#   torch-2.12.1+cpu-cp311-cp311-manylinux_2_28_aarch64.whl) so pip
#   installs the pure CPU build directly — no NVIDIA CUDA libraries
#   are pulled in and torch imports cleanly on devices without a GPU.
#
# • numpy and all other packages are resolved from PyPI as normal via
#   --extra-index-url (PyTorch's index is added alongside, not instead).
#
# • matplotlib is intentionally excluded.  The bridge runs headless; all
#   visualisation is done on the Mac before deployment.  If you need to run
#   evaluate.py on the Pi, install matplotlib separately:
#     pip install matplotlib
#
# • No torchvision / torchaudio / MPS extras needed.

# ── Direct pip to the CPU-only wheel index (has aarch64 from torch 2.6+) ────
--index-url https://download.pytorch.org/whl/cpu
--extra-index-url https://pypi.org/simple

# ── Core deep-learning runtime (CPU-only aarch64 wheel ~500 MB) ─────────────
torch>=2.6.0

# ── Numerical / signal-processing core ───────────────────────────────────────
numpy>=1.24.0

# ── Numerical / signal-processing core ───────────────────────────────────────
numpy>=1.24.0
